# Bitácora

La **memoria del proyecto**. Las decisiones viven aquí, no en el chat. Si chat y bitácora divergen, **gana la bitácora**. No se borran las entradas viejas (son evidencia de proceso).

## Nombrado
```
YYYY-MM-DD_<tema-corto-kebab>_<agente>.md
```
Ejemplos: `2026-06-02_kickoff_talaia.md`, `2026-06-05_spec-pipeline_talaia-a-sondeig.md`.

## Plantilla
```markdown
# <Título breve>

**Fecha:** YYYY-MM-DD
**Autora:** <agente o humana>
**Para:** <destinataria si aplica>
**Tema:** <una frase>
**Status:** <propuesta / decisión / avance / bloqueo / aprendizaje / spec / handoff>

## Contexto
Qué situación motivó la entrada.

## Qué hicimos / decidimos
Lo concreto.

## Por qué
El razonamiento (lo que más vale dentro de N semanas).

## Pendiente
- [ ] acción

## Enlaces
- ...
```

## Reglas cortas
1. Si una decisión se discute >10 min, merece entrada.
2. Mejor muchas entradas cortas que una larga.
3. Si bloqueas a otra agente, entrada tipo "bloqueo".
4. Trivial (typo, fix de un parámetro): commit message, no bitácora.
